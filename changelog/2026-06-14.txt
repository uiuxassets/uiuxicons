Fixed tree-shaking in @uiuxicons/react and @uiuxicons/vue. Both packages now ship one module per icon (instead of a single pre-bundled file) and mark each icon as side-effect-free, so bundlers include only the icons you import. Apps using a handful of icons no longer pull in the full set. Added optional per-icon subpath imports (e.g. @uiuxicons/react/icons/gear). Bumped both packages to 0.1.2.
